# YOLO 核心理论

## 1. YOLO 的名字

**YOLO = You Only Look Once**

只看一次图片就能检测出所有目标。对比两阶段检测器（如 Faster R-CNN）需要先生成候选区域再分类，YOLO 一步到位。

## 2. 核心思想：网格划分

```
将图片分成 S×S 个网格（如 7×7）
每个网格负责检测中心落在该格子内的目标
每个网格预测 B 个 bounding box + 置信度 + C 个类别概率
```

```
┌───┬───┬───┬───┬───┬───┬───┐
│   │   │   │   │   │   │   │
├───┼───┼───┼───┼───┼───┼───┤
│   │   │ ┌─┼───┼───┐   │   │
├───┼───┼─┤█│███│█├─┼───┼───┤
│   │   │ └─┼───┼───┘   │   │
├───┼───┼───┼───┼───┼───┼───┤
│   │   │   │   │   │   │   │
├───┼───┼───┼───┼───┼───┼───┤

```

## 3. YOLOv8/v11 架构

```
输入图片 (640×640×3)
       │
       ▼
┌─────────────┐
│   Backbone   │  特征提取
│   (C2f)     │  从像素中提取语义特征
└──────┬──────┘
       │
       ▼
┌─────────────┐
│    Neck      │  特征融合
│  (FPN+PAN)  │  多尺度特征聚合
└──────┬──────┘
       │
       ▼
┌─────────────┐
│    Head      │  检测头
│ (Decoupled) │  输出: box + class
└──────┬──────┘
       │
       ▼
  预测结果
  (boxes, scores, classes)
```

### Backbone（骨干网络）
- 作用：从原始像素中提取特征
- 演化：Darknet → CSPDarknet → C2f
- C2f（Cross Stage Partial with 2 convolutions）：更高效的特征提取

### Neck（颈部）
- 作用：融合不同尺度的特征
- 问题：小目标在深层特征中信息丢失
- 方案：FPN（自上而下）+ PAN（自下而上）双向融合

### Head（检测头）
- 作用：从特征图生成最终预测
- 演化：Anchor-based → Anchor-free → DFL
- YOLOv8 使用 Decoupled Head：分类和回归分开处理

## 4. 多尺度检测

```
P3 (80×80) → 检测小目标 (如远处的人)
P4 (40×40) → 检测中等目标 (如车)
P5 (20×20) → 检测大目标 (如建筑)
```

## 5. 关键概念

### IoU (Intersection over Union)
```
IoU = 交集面积 / 并集面积

  ┌───────────┐
  │  预测框    │
  │     ┌─────┼─────┐
  │     │交集 │     │
  └─────┼─────┘     │
        │   真实框  │
        └───────────┘

IoU > 0.5 通常认为是好的检测
```

### NMS (Non-Maximum Suppression)
```
输入：多个重叠的预测框 + 置信度
过程：
  1. 按置信度排序
  2. 取最高分的框加入结果
  3. 删除与它 IoU > 阈值的框
  4. 重复
输出：去重后的检测结果
```

### Anchor（锚框）
- 预定义的参考框，不同大小和比例
- 模型预测相对于 anchor 的偏移量
- YOLOv8 已经是 anchor-free，直接预测框坐标

## 6. Loss 函数

YOLOv8 的 loss 由三部分组成：

```
Loss = λ₁·box_loss + λ₂·cls_loss + λ₃·dfl_loss
```

| Loss | 含义 | 作用 |
|------|------|------|
| box_loss | CIoU Loss | 让预测框更准确地匹配真实框 |
| cls_loss | BCE Loss | 让分类更准确 |
| dfl_loss | Distribution Focal Loss | 更精确地回归边界 |

## 7. 评估指标

### mAP (mean Average Precision)
```
对每个类别：
  1. 画 PR 曲线 (Precision vs Recall)
  2. 计算曲线下面积 = AP

mAP = 所有类别 AP 的平均值

mAP@0.5   : IoU 阈值固定 0.5
mAP@0.5:0.95: IoU 阈值从 0.5 到 0.95，步长 0.05，取平均
```

## 8. 训练策略

### 迁移学习
```
预训练模型 (COCO 118k 图片)
       │
       ▼
冻结 Backbone → 只训练 Head
       │
       ▼
微调全部层 (小学习率)
```

### 数据增强
- **Mosaic**: 4 张图拼成 1 张，增加上下文多样性
- **MixUp**: 两张图混合，增加正则化
- **HSV**: 色调/饱和度/亮度随机变化
- **翻转/旋转/缩放**: 几何变换

## 9. YOLO 演化历史

```
YOLOv1 (2016) → 开创性工作，7×7 网格
YOLOv2 (2017) → 引入 anchor、BatchNorm
YOLOv3 (2018) → 多尺度检测、FPN
YOLOv4 (2020) → Mosaic 增强、CSPDarknet
YOLOv5 (2020) → ultralytics 实现，工程化
YOLOv7 (2022) → E-ELAN 结构
YOLOv8 (2023) → Anchor-free、Decoupled Head
YOLOv11 (2024) → 最新版本，持续优化
```
